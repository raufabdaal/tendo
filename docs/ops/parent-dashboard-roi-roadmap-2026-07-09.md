# Parent Dashboard / ROI Roadmap — 2026-07-09

## Why this matters

In the new premium parent-funded direction, the paying user is the parent. The parent needs to see that Tendo is worth paying for.

A parent dashboard is now more important than a teacher dashboard.

---

## Parent dashboard goal

A parent should quickly answer:

1. Is my child using Tendo?
2. What has my child studied?
3. Where is my child struggling?
4. What should my child do next?
5. Am I getting value for my monthly payment?

---

## MVP parent dashboard sections

### 1. This week

- minutes studied or sessions completed
- topics studied
- questions answered
- papers attempted

### 2. Strengths

- strongest subject
- strongest topics
- recent improvement

### 3. Needs support

- weak topics
- repeated mistakes
- subjects with low activity

### 4. Recommendation

Simple parent-friendly next action:

- “Do 10 Maths questions before Friday.”
- “Revise Letter Writing; score is below target.”
- “Try one past paper section this weekend.”

### 5. Progress trend

Simple progress indicator, not analytics overload.

---

## Parent language principles

Use plain language:

- “Needs support” not “low performance segment”.
- “Practised 4 topics this week” not “engagement metric”.
- “Recommended next step” not “AI intervention”.

---

## Data required later

A real parent dashboard requires backend-auth data:

- student account
- parent/guardian link
- topic progress
- practice answers
- paper attempts
- teacher-set question marks
- recommendations

For now, localStorage can support a prototype, but real paid usage needs cross-device sync.

---

## Priority

Parent dashboard should come after:

1. P7 content is premium-complete enough.
2. Student practice/study loop is stable.
3. Backend auth direction is chosen.

But its UX should be designed early because it shapes what data we collect.
